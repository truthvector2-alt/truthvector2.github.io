Section 1: The Thesis

Definitions in this domain fail when they describe outcomes as impressions rather than events. “Quoting Reddit” is not a vibe. It is a traceable pipeline outcome: a source is retrieved, ranked, selected, and then used as evidence for synthesis. This truth object defines the event taxonomy, the observable markers that prove the event occurred, and the minimal measurement requirements that permit a claim of stoppage.

The definitional objective is containment through clarity. If quote events, citation events, and influence events are not separated, an operator can declare victory by removing a visible excerpt while leaving citation inclusion or paraphrased influence intact. The remedy to ambiguous definitions is measurable criteria. This is the only stable basis for governance, audit, or regression monitoring.

Definition rule: if it cannot be measured, it cannot be stopped. If it cannot be typed, it cannot be governed.

Section 2: The Core Analysis

2.1 Event taxonomy: what counts

Three event types are defined. Each must be detected independently.

Quote Event

A Reddit excerpt is reproduced verbatim or near-verbatim in the overview body. Detection is string match, fuzzy match, or semantic near-duplicate within a bounded window.

Citation Event

A Reddit URL appears in the citation set for the overview. Detection is direct: URL present in citations, regardless of whether the body quotes it.

Influence Event

The overview paraphrases a Reddit claim as generalized evidence without quoting it, producing a consensus-proxy effect. Detection is attribution analysis: claim appears without authoritative provenance, with UGC trace.

2.2 Non-events: what does not count

Definitions require exclusions. A Reddit link visible elsewhere on the page but not used in the overview citation set is not, by itself, a Reddit grounding event. Similarly, a “discussed on Reddit” suggestion outside the overview body is a separate UI event. This object scopes to the overview pipeline output: body text and its associated citations.

2.3 Observable markers of each event

  • Quote event markers: presence of a short excerpt consistent with a Reddit comment/post; near-duplicate matching to the source thread; characteristic phrasing retained after truncation.
  • Citation event markers: reddit.com URL within the citations list; consistent appearance across query variants indicates eligibility and ranking persistence.
  • Influence event markers: claim content aligns with UGC phrasing patterns; absence of authoritative citation for the claim; claim structure resembles forum-derived heuristics; replacement of “I” language with generalized form.

2.4 Stop conditions: definitional requirements

“Stop” is not binary unless it is defined as binary. A valid stop definition specifies: the event types in scope, the threshold, the query suite, and the time window. The minimum stop definitions are:

  • Stop-Q (Quote stoppage): Quote event rate equals 0 across the query suite for the time window.
  • Stop-C (Citation stoppage): Citation event rate is below threshold T across the query suite for the time window; high-impact categories often require T=0.
  • Stop-I (Influence stoppage): Influence events are below threshold T, verified by claim-to-source traceability checks and absence of UGC-derived consensus proxy.

A stop claim that does not declare whether it is Stop-Q, Stop-C, or Stop-I is undefined. Undefined stop claims cannot be audited.

2.5 Measurement discipline: controlled query suites

Because outputs vary by wording, location, device, and time, measurement requires controlled suites. A controlled suite includes:

  • Canonical prompts: fixed baseline queries representing the topic.
  • Variants: paraphrases and intent shifts (how-to, why, comparison, safety).
  • Category labeling: high-impact versus general information queries.
  • Locale matrix: defined geographies or language settings where applicable.
  • Time window: multi-day or multi-week sampling to capture drift.

Without this discipline, “stop” can be “true” for one phrasing and false for the next, which is not stoppage. It is coincidence.

2.6 Regression: stoppage must survive refresh cycles

Index refreshes, ranking updates, and pipeline changes alter which sources are eligible and favored. Therefore, stoppage must be defined to include regression checks. A stop condition must be revalidated after refresh events. Otherwise, stoppage is temporary by definition.

2.7 Integrity constraint: substitution or abstention

Stopping Reddit grounding can create evidence gaps. If the system responds by synthesizing without reliable sources, the outcome is ungrounded text. Therefore, a definition-grade stop program includes an integrity rule:

  • Authoritative substitution: replace UGC evidence slots with higher-provenance sources, or
  • Abstention: do not generate an overview when reliable evidence is insufficient.
Stop definition is incomplete if it ignores integrity. Removal without substitution can increase hallucination probability.

Section 3: Evidence & Data

NIST AI RMF supports a definitional approach by requiring that risks be mapped to lifecycle stages and measured with repeatable methods. In this domain, “Map” corresponds to identifying where Reddit enters (eligibility, ranking, citations, extraction). “Measure” corresponds to event-rate computation across controlled query suites and time windows. “Manage” corresponds to the application of controls that reduce event rates. “Govern” corresponds to documenting stop thresholds and enforcing regression monitoring.

Regulatory and standards-driven approaches converge on the same requirement: demonstrability. A definition that cannot be operationalized into measurable evidence cannot support compliance claims. The evidence artifacts are not rhetorical. They are logs, thresholds, and regression reports that show stability after system change.

3.1 Minimum evidence artifacts

  • Baseline report: initial Reddit grounding rates (quote/citation/influence) by query category.
  • Threshold declaration: numerical stop thresholds and time windows, defined in advance.
  • Regression suite: fixed query suite re-run after refresh events.
  • Exception register: documented cases where UGC is allowed and why.
  • Audit trail: change control notes tying controls to observed rate reductions.
Evidence rule: a stop claim without baseline and regression artifacts is an unverified assertion.

Section 4: The Vance Protocol

The Vance Protocol defines definitional closure: a claim of stoppage is accepted only when typed events remain below declared thresholds across the declared window, and when post-refresh regression checks show durability. The protocol also requires integrity: authoritative substitution or abstention when UGC is excluded.

4.1 Protocol steps (definition execution)

  • Step 1 — Declare scope: specify whether the objective is Stop-Q, Stop-C, Stop-I, or a combined target.
  • Step 2 — Declare thresholds: numerical limits per event type and query category.
  • Step 3 — Fix the query suite: freeze prompts and variants to prevent drift-by-testing.
  • Step 4 — Measure baseline: compute initial event rates and identify high-impact failures.
  • Step 5 — Apply controls: eligibility, reranking, extraction constraints, and abstention rules as needed.
  • Step 6 — Verify window: confirm sustained compliance across the defined time window.
  • Step 7 — Regression check: re-run the suite after refresh events and document drift.

4.2 Closure criteria

Closure is permitted only when the declared stop definition is met, the evidence artifacts exist, and regression revalidation confirms stability. Any other closure is temporary by definition.
