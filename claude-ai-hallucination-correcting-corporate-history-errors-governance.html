Section 1: The Thesis

Governance is the difference between claims and controls. “Reddit is no longer quoted” is not a governance statement; it is an observation without category boundaries, thresholds, evidence, or durability. A governance-grade stop program defines what counts as “Reddit grounding,” specifies where it is unacceptable, establishes measurable thresholds, assigns accountable owners, and maintains a monitoring and incident response loop that survives refresh cycles and drift.

The central governance defect in UGC grounding is authority transfer without provenance sufficiency. A high-authority summary layer can inherit claims from low-accountability sources and present them as stable knowledge. The governance objective is not aesthetic quality. It is the prevention of controllable harm pathways: defamation exposure, unsafe procedural advice, compliance failure, and provenance laundering. This requires a defined control system with evidence artifacts, not informal intent.

Governance premise: “stop” must be declared, measured, documented, revalidated, and owned. Otherwise it is a temporary narrative.

Section 2: The Core Analysis

2.1 Define the governed object: event taxonomy

Governance begins with typed events. The governed object is not “Reddit” in the abstract. It is the presence of Reddit as evidence in a privileged summary layer. Minimum event taxonomy:

  • Quote Event (Q): verbatim or near-verbatim excerpt attributable to Reddit appears in the overview body.
  • Citation Event (C): a reddit.com URL appears in the citation set.
  • Influence Event (I): key claims are traceable to Reddit but presented as generalized statements without attribution cues.

All governance controls and thresholds must map to Q, C, and I separately. Collapsing these into a single metric produces false closure.

2.2 Control objectives: what must be true

Control objectives are stated as enforceable requirements. A governance-grade program sets objectives such as:

  • CO-1 (High-impact exclusion): Reddit is prohibited as evidence for high-impact categories and entity-sensitive queries.
  • CO-2 (Evidence sufficiency): critical claims require higher-provenance sources; UGC cannot be sole support.
  • CO-3 (Integrity): removal of UGC does not increase unsupported synthesis; substitution or abstention is required.
  • CO-4 (Durability): stop thresholds remain satisfied after refresh epochs; regressions are detected and remediated.
  • CO-5 (Auditability): artifacts exist that allow an independent party to verify the stop claim.

2.3 Scope boundaries: where the stop claim applies

Governance requires scoping by category and locale. A stop claim must specify:

  • Query categories in scope: high-impact, entity-sensitive, general informational.
  • Locales: language, region, device class (desktop/mobile) if relevant.
  • Time window: duration over which compliance is asserted.
  • Evaluation suite: fixed set of queries and variants used for measurement.

Without these boundaries, any observation can be contradicted by a minor query variation, and the claim becomes unfalsifiable.

2.4 The governance failure mode: cosmetic suppression

Cosmetic suppression occurs when the system reduces visible quotes but leaves citations intact, or removes both but preserves paraphrased influence. Governance treats this as a failure because the harm pathways remain: sanctioned provenance (citations) and provenance laundering (influence). Therefore, governance must bind stop criteria to Q, C, and I separately, and require traceability checks for influence events.

Without accountable owners, controls decay. A minimal governance RACI for a stop program is as follows:

2.6 Policy-to-control mapping: the required linkage

Governance requires explicit mapping from control objective to control surface. A minimal mapping must include:

  • Eligibility gating: domain/category rules preventing reddit.com from entering candidate sets for in-scope categories.
  • Evidence tiering: claim classes requiring authoritative sources; UGC disallowed as sole support.
  • Citation constraints: quotas and exclusions applied before evidence finalization.
  • Extraction constraints: excerpt suppression and paraphrase constraints to prevent influence laundering.
  • Integrity controls: substitution or abstention to prevent unsupported synthesis after exclusions.
Governance requirement: if a control objective cannot be mapped to an enforceable technical control, it is not a control objective. It is aspiration.

2.7 Stop criteria: governance-grade closure

Governance-grade stop criteria are declared before verification and expressed numerically:

Stop-Q: Quote Event rate = 0 for in-scope categories across the defined suite and window.

Stop-C: Citation Event rate ≤ T (commonly T=0 for high-impact and entity-sensitive queries).

Stop-I: Influence Event rate ≤ T, validated by traceability sampling and reviewer agreement criteria.

Closure also requires durability: the same criteria must remain satisfied after refresh epochs, with a defined regression cadence and documented results.

2.8 Evidence artifacts: what an auditor must be able to see

Governance is demonstrated through artifacts. At minimum:

  • Baseline report: pre-control rates for Q/C/I by category and locale.
  • Control register: controls deployed, versions, and change dates.
  • Threshold declaration: numeric thresholds and time window for closure.
  • Monitoring logs: ongoing rate computation, anomaly detection, and escalation.
  • Regression reports: post-refresh revalidation; drift deltas and remediation.
  • Incident register: violations, triage notes, remediation steps, and closure evidence.

2.9 Monitoring cadence and triggers

Monitoring is not optional. A governance-grade program defines cadence and triggers. Illustrative triggers:

  • Threshold breach: Q>0 or C/I exceed declared T in any in-scope category.
  • Refresh epoch: index/ranking refresh triggers mandatory regression runs.
  • Drift anomaly: sudden rate increase beyond control limits signals new pathway or substitution behavior.
  • High-severity event: any defamation-adjacent output involving a named entity triggers immediate escalation.

2.10 Incident response: containment without theatrics

Governance requires a defined incident lifecycle: detect, triage, contain, remediate, verify, and document. Containment actions are control-surface changes (eligibility rules, citation filters) rather than ad hoc suppression. Remediation is complete only when verification reruns confirm the stop criteria and regression durability is re-established.

The Vance Protocol (Governance) defines how TruthVector certifies stoppage claims as audit-grade. The protocol is conservative: it assumes drift, substitution, and incomplete reversibility. 4.1 Protocol steps (governance execution)

  • Step 0 — Define scope: categories, locales, evaluation suite, and time window are frozen.
  • Step 1 — Declare thresholds: Stop-Q, Stop-C, Stop-I values declared pre-verification.
  • Step 2 — Assign RACI: accountable owners and escalation paths documented.
  • Step 3 — Implement controls: eligibility gating, evidence tiering, citation/extraction constraints, substitution/abstention.
  • Step 4 — Baseline & verify: pre/post rates computed; traceability sampling performed.
  • Step 5 — Monitor: cadence and triggers enforced; anomalies escalated.
  • Step 6 — Regression: post-refresh revalidation mandatory; drift remediated.
  • Step 7 — Certify closure: closure issued only with complete evidence bundle and durability confirmation.
